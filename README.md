# Listing Sentinel (Local-Only)

フリマ・中古サイトなどの出品ページを、ボタン1つで簡易チェックする Chrome 拡張（Manifest V3）。
判定もログも**すべて端末内**。外部通信は一切しない。

## Purpose

「この出品、ちょっと怪しいかも？」を購入前に**1行で**目で確認するための、軽い参考ツール。
最終判断は人間がする前提の「参考メモ」であって、自動ブロックや断定はしない。

## Stack

- Chrome 拡張 / Manifest V3
- 素の HTML + CSS + JavaScript（ビルド不要・依存ライブラリゼロ）
- 保存は `chrome.storage.local`（端末内のみ）

## Setup / Dev

ビルド不要。Chrome に読み込むだけ。

1. Chrome で `chrome://extensions` を開く
2. 右上の **デベロッパーモード** を ON
3. **パッケージ化されていない拡張機能を読み込む** → このフォルダ（`mvp-anti-scam`）を選択
4. 対象の出品ページを開いて、ツールバーの拡張アイコンをクリック

> ⚠️ 危険度：低。自分の PC で自分用に使う範囲なら実害なし。
> ただしアイコンを押したページの内容を読む権限を使う（押した時だけ・通信なし）。**配布はしない前提**。

## Structure

```
mvp-anti-scam/
├─ manifest.json   # 拡張の設定。クリック時のみ動く最小権限
├─ scan.js         # ページから出品情報を1つ取り出す（注入される）
├─ popup.js        # 判定ロジック・保存・印刷/CSV
├─ popup.html      # ボタンUI
├─ print.html      # A4サマリー
├─ print.js        # A4サマリーの中身生成
├─ styles.css      # 配色（水色/白/グレー）・フォント
└─ README.md
```

## Features

アイコンを押すと、今開いているページを **その時だけ1回** 読み取って判定する。

- **価格が安すぎる疑い**：同カテゴリで端末内にためた価格の中央値より 30% 以上安い
- **新規アカウントの疑い**：出品者の登録日数が 30 日未満（日付が読み取れた時だけ）
- 判定は一行表示：「疑わしい / やや注意 / 概ね問題なし / 判定材料が少ない」＋理由の箇条書き
- **A4印刷**：直近の判定を1枚にまとめて印刷
- **CSV出力**：たまったログを書き出し（Excel で開いても文字化けしない）
- **誤検知ボタン**：外していたら `fp` ラベルを付けて、後でしきい値調整の材料にする

### 元案から意図的に変えた点（正直メモ）

- **常時監視 → クリック時だけ**に変更。全サイトを常に読むのは重く、出品以外のゴミも溜まるため。プライバシーと安全性のため。
- **画像pHash（重複画像判定）は今回入れていない**。フリマの画像は別ドメイン配信が多く、ブラウザのCORS制約で Canvas が「汚染」され `getImageData` が例外になり、実環境ではほぼ機能しないため。7日運用が回り始めてから別途検討する。
- 価格抽出は「CSSクラスの当てずっぽう」より **JSON-LD（schema.org）優先**にして、サイト横断での精度を上げた。

## Data Structure

`chrome.storage.local` に以下を保存：

```jsonc
{
  "median:<category>": [1200, 1500, ...],   // 同カテゴリの直近価格（最大100件）
  "lastScan": { /* 直近の判定 */ },
  "lastListing": { /* 直近に読んだ出品情報 */ },
  "telemetry": [
    {
      "event": "scan",
      "listing_id": "https://example.com/items/123",
      "host": "example.com",
      "title": "...",
      "heuristics": ["price_anomaly", "seller_new"],
      "verdict": "suspicious",
      "ui_version": "0.2.0",
      "ts": 1716998400000,
      "meta": { "price": 1000, "median": 1500, "samples": 12, "seller_age_days": 5 },
      "label": "fp"   // 誤検知ボタンを押すと付く
    }
  ]
}
```

## チューニング計画（7日）

しきい値は `popup.js` 冒頭の `SETTINGS` だけ見れば変えられる。

- Day1–2：価格中央値が安定するまで普段見るページで数を貯める
- Day3–4：`priceDropRatio`（0.7 を 0.6〜0.75 で）を調整
- Day5–6：`sellerNewDays`（30 を 21 / 45 で）を調整
- Day7：CSV を見て、誤検知（`fp`）が多い項目のしきい値を再設定
